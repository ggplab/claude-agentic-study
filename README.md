각 챕터를 읽고 **자기에게 필요한 걸 하나씩 직접 만들어 본 뒤**, 무엇을 만들었는지와 시행착오를 함께 기록·공유하는 공간입니다.

## 운영 방식 (회차당 60분)

> 정기 모임: **매주 일요일 밤 9:30 (21:30)** · 회차당 60분

| 시간 | 내용 |
|------|------|
| 0–20분 | **챕터 리드**가 핵심 요약 + 진행 (개념 압축, 질문 받기) |
| 20–45분 | **각자 "핵심 개념 1개"** 공유 — 1인 5~6분 (내용 리마인드 + 나에게 어떤 의미였나) |
| 45–60분 | **LinkedIn 초안** 그 자리에서 작성 → 끝나고 **24시간 내 발행** |

규칙 세 가지:
1. **각자 서로 다른 개념을 고른다** — 회차 시작 때 리드가 30초 교통정리. 겹치지 않게 → 피드 중복 방지 + 합치면 그 챕터를 통째로 커버
2. **"의미"는 책 요약이 아니라 내 작업/프로젝트에 어떻게 쓸지로** 적는다 — 본인 인사이트가 콘텐츠가 됨
3. **모임 마지막 10분에 그 자리에서 `chapters/chNN/README.md`에 기록하고 커밋** — 따로 정리하면 안 한다

> 리드는 챕터마다 로테이션. 발표 부담은 가볍게(개념 정리만), 비중은 각자 공유 + 포스팅에.

### 주간 흐름 (두 규칙)

1. **참여자**: 모임 전까지 그 챕터의 `chapters/chNN/README.md`에 **기본글**(고른 개념 / 리마인드 / 의미)을 쓰고, **LinkedIn에 포스팅**한다. → Claude Code에서 `/study-chapter <챕터번호>`에 읽기 메모를 붙여넣으면 개념정리 + 본인 클로드 시스템 적용예시 실측 + 기본글 + LinkedIn 초안까지 만들어 정리본만 푸시해준다. 원본 메모·초안은 `_drafts/`(gitignore)에 로컬로만 남는다. (스킬: [`.claude/skills/study-chapter`](.claude/skills/study-chapter/SKILL.md))
2. **리더**: **일요일 오후**에 참여자들이 올린 글을 보고 **발표 순서를 조정**한다. → Claude Code에서 `/order-session <챕터번호>` 실행하면 자동으로 글을 읽어 중복 점검 + 순서 제안 + README 기록까지 해준다. (스킬: [`.claude/skills/order-session`](.claude/skills/order-session/SKILL.md))

```
[참여자] 기본글 작성 + LinkedIn 포스팅
            ↓ (일요일 오후)
[리더] /order-session 으로 순서 조정
            ↓
[모임 60분] 리드 요약 → 각자 발표 → 회고/커밋
```

### LinkedIn 포스팅 톤
- 대괄호 제목 X, 정제된 존댓말
- 번호 리스트로 구조화, 해시태그 X
- 개념 설명 + 본인 경험/적용을 한 덩어리로
