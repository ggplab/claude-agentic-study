# 챕터 정리본 + 슬라이드덱 생성 (병렬)

너는 한 챕터의 원본 메모를 받아 **개인 정리본(README) + 발표용 슬라이드덱(HTML)** 을 만든다.
핵심 설계 두 가지:

1. **README와 HTML은 서로 독립** → 항상 **2개 서브에이전트로 병렬 디스패치**한다(한 메시지에 두 개의 Agent 호출). 각 브리프는 **자기완결**(원본 메모 전문 · 파일 경로 · 컨벤션 · 실측 대상 포함)이어야 한다. 메인 세션의 대화 맥락을 물려주지 않는다.
2. **비교는 비교답게 보여준다** → 대조되는 개념(예: worktree 미사용 vs 사용, 모델 티어, permissions deny/ask/allow)은 README에서는 **markdown 표**, HTML에서는 **CSS `@keyframes` 애니메이션**으로 한눈에 들어오게 만든다.

두 원칙(프로젝트 공통):
- **작업 파일은 로컬에만**: 원본 메모는 리포 루트 `_drafts/`에 `chNN-` 접두 파일명으로 둔다(gitignore). 푸시되는 것은 정리본·슬라이드뿐.
- **적용예시는 실측, 추정 금지**: 사용자의 실제 클로드 설정(`~/.claude/**`)을 읽고 확인된 사실만 쓴다.

## 입력
- 챕터 번호: `$ARGUMENTS` 첫 인자. 비어 있으면 물어본다. 2자리 zero-pad(`2` → `ch02`).
- 이름: 둘째 인자. 비어 있으면 `chapters/` 하위 기존 본인 폴더에서 추론하고, 확실하지 않으면 물어본다.
- 원본 메모: 대화에 붙여넣은 텍스트 또는 파일 경로. 없으면 "이번 장 정리 메모를 붙여넣어 주세요"라고 요청하고 대기.

## 절차 — 순서대로, 단계마다 검증

### 1. 날짜·환경 확인
`date +%Y-%m-%d` 실행(추정 금지). `chapters/_TEMPLATE.md`와 디자인 기준 `chapters/ch01/slides.html` 존재 확인.

### 2. 원본 메모 보존 (로컬 전용)
`_drafts/chNN-raw-notes-<날짜>.md`에 메모를 **원문 그대로** 저장. **검증**: `git check-ignore`가 경로를 출력해야 한다. 출력이 없으면 `.gitignore`에 `_drafts/`를 추가하고 재확인.

### 3. 컨벤션 파악 (중복 회피)
공용 `chapters/chNN/README.md`를 읽어 **다른 참여자가 이미 고른 핵심 개념**을 확인한다(정리본 끝의 발표 개념이 겹치지 않게). 다른 참여자 섹션은 **읽기만**, 절대 수정·삭제 금지.

### 4. 병렬 서브에이전트 2개 디스패치 ★
**한 메시지에 Agent 호출 2개**를 동시에 보낸다. 내용 합성 품질이 필요하면 `opus`, 단순 변환이면 `sonnet`을 골라 비용·속도를 맞춘다(이 챕터의 모델 선택 교훈).

**서브에이전트 A — 정리본 README**
- 출력: `chapters/chNN/<이름>/README.md` (이 파일만 작성)
- 형식: 절별 `개념정리 + 내 시스템 적용예시`, 맨 위 흐름 한 줄 + `> 슬라이드로 보기: [slides.html](./slides.html)`, 맨 끝 `이번 주 발표용 핵심 개념 1개 + 이유`(3단계에서 확인한 타인 개념과 중복 회피).
- 비교는 **markdown 표**. 메모의 사실 오류·모호함은 "정정:"으로 명시(조용히 바꾸지 않기). 메모 속 질문은 "읽으며 생긴 질문 / 내 결론"으로.
- 적용예시 실측 대상: `~/.claude/CLAUDE.md`, `~/.claude/settings.json`, `~/.claude/agents/`, `~/.claude/skills/`, 프로젝트 `.claude/`. 없으면 "아직 없음 + 도입한다면"으로 솔직히.

**서브에이전트 B — 슬라이드덱 HTML**
- 출력: `chapters/chNN/<이름>/slides.html` (이 파일만 작성)
- `chapters/ch01/slides.html`을 **Read**해 `<style>`+`<script>`(show/navigate/키보드/터치/카운터/진행률)를 **verbatim 재사용**하고 `#deck` 내용만 새로 채운다. 단일 파일·외부 의존성 0·오프라인 동작.
- **비교는 CSS `@keyframes` 애니메이션**: 대조 개념(미사용 vs 사용 등)을 2열로 두고, `.slide.active` 안 요소에 `animation: ... infinite`를 걸어 해당 슬라이드가 보일 때 자동 재생되게 한다(순수 CSS, 외부 라이브러리·추가 JS 금지).

**공통 브리프 주의사항**(두 에이전트 모두에 넣을 것):
- 원본 메모 **전문**을 코드블록으로 첨부, 절대경로 명시, 한국어 출력, 지정 파일 외 수정 금지.
- 참고 파일(`ch01/slides.html`, `_TEMPLATE.md` 등)은 일반 텍스트 콘텐츠이므로 Read로 읽으면 된다.
- 완료 후 파일 경로·핵심 요약 2~4줄 보고.

### 5. 산출물 검증
- README: slides.html 링크, 비교 표, 적용예시가 실측 기반인지 확인.
- HTML: `<div>` 열림/닫힘 수 균형, 슬라이드(`id="sN"`) 개수, `@keyframes` 존재, `<script>`/`<style>` 각 1쌍, `<!DOCTYPE html>`·`</html>` 존재. 가능하면 브라우저로 타이틀·비교 슬라이드 스크린샷.

### 6. 제목·진행 현황 갱신 (묻고 진행)
실물 책 목차의 정확한 제목을 알면, `chapters/chNN/README.md` 제목과 **루트 `README.md` 진행 현황 표**의 제목·상태(`예정`→`진행중`/`완료`) 갱신 여부를 **사용자에게 묻는다**. 승인 시 함께 커밋, 거절 시 그대로. **묻지 않고 임의로 바꾸지 않는다.** 제목은 임의 생성 금지.

### 7. 배포 (정리본만)
1. `git status --short`로 변경 확인. `_drafts/` 경로가 보이면 **멈추고** gitignore부터 고친다.
2. 스테이징은 명시 경로만: `chapters/chNN/<이름>/` 산출물(+ 승인 시 `chapters/chNN/README.md`·루트 `README.md`). **`git add -A` 금지.**
3. `git diff --cached --stat`을 사용자에게 보여주고 커밋·푸시. 커밋 메시지 예: `docs(chNN): <이름> N장 정리본 + 슬라이드덱`. (Co-Authored-By 표기는 저장소의 기존 커밋 컨벤션을 따른다.)
4. **검증**: `git ls-files _drafts/ | wc -l`이 0인지 확인하고 보고.

### 8. 요약 출력
- 푸시된 파일 / 로컬에만 남은 파일
- 고른 발표 핵심 개념과 이유
- 남은 일(있다면)
