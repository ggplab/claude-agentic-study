# 주간 챕터 정리 (참여자용)

너는 스터디 참여자의 주간 흐름을 자동화한다: **읽기 → 개념정리 → 내 클로드 시스템 적용예시 파악 → 포스팅 초안 → 정리본만 푸시**.

핵심 원칙 두 가지:
1. **작업 파일은 로컬에만**: 원본 메모·포스팅 초안은 `chapters/chNN/<이름>/_drafts/`에 둔다 (`.gitignore`의 `_drafts/`로 커밋 제외). 푸시되는 것은 정리본과 챕터 README 기본글뿐.
2. **적용예시는 실측, 추정 금지**: 사용자의 실제 설정 파일을 읽고 확인된 사실만 쓴다.

## 입력
- 챕터 번호: `$ARGUMENTS` 첫 인자. 비어 있으면 물어본다. 2자리 zero-pad (`3` → `ch03`).
- 이름: 둘째 인자. 비어 있으면 `chapters/` 하위 기존 본인 폴더에서 추론하고, 확실하지 않으면 물어본다.
- 원본 메모: 사용자가 대화에 붙여넣은 텍스트 또는 파일 경로. 없으면 "이번 장에서 정리한 메모를 붙여넣어 주세요"라고 요청하고 대기.

## 절차 — 순서대로 실행, 단계마다 검증

### 1. 날짜·환경 확인
`date +%Y-%m-%d` 실행 (날짜 추정 금지). `chapters/_TEMPLATE.md` 존재 확인.

### 2. 원본 메모 보존 (로컬 전용)
`chapters/chNN/<이름>/_drafts/raw-notes-<날짜>.md`에 메모를 **원문 그대로** 저장한다 (오타 포함, 가공 금지).
**검증**: `git check-ignore chapters/chNN/<이름>/_drafts/raw-notes-<날짜>.md`가 경로를 출력해야 한다. 출력이 없으면 `.gitignore`에 `_drafts/`를 추가한 뒤 재확인하고 다음 단계로.

### 3. 개념정리
메모의 절 구조를 유지하면서 각 절을 정리한다.
- 메모에 사실 오류나 모호한 부분이 있으면 **고쳐 쓰되 정정했다고 명시**한다 (조용히 바꾸지 않는다).
- 메모 속 질문(예: "~도 동일?")은 "읽으며 생긴 질문과 내 결론" 형태로 답을 달아 정리한다.

### 4. 적용예시 실측
각 절마다 사용자의 클로드 시스템에서 **실제로 확인한** 예시를 1개 이상 찾는다. 조사 대상:
- `~/.claude/settings.json` (permissions allow/deny, hooks, plugins)
- `~/.claude/skills/`, `~/.claude/agents/`, `~/.claude/rules/`
- `~/CLAUDE.md` 및 `@import` 체인, 프로젝트 `.claude/`
파일을 읽지 않고 기억으로 쓰는 것 금지. 해당 개념의 예시가 시스템에 없으면 "아직 없음 + 도입한다면 어디에"로 솔직하게 쓴다.

### 5. 정리본 작성 (푸시 대상)
`chapters/chNN/<이름>/README.md`에 작성: 절별로 **개념정리 + 내 시스템 적용예시**, 마지막에 **이번 주 발표용 핵심 개념 1개 선택**과 선택 이유 (다른 참여자와 겹치지 않을 만한 것 우선 — 챕터 README에 이미 올라온 다른 사람 개념을 확인해 회피).

### 6. 챕터 README 기본글 (푸시 대상)
`chapters/chNN/README.md`가 없으면 `chapters/_TEMPLATE.md`를 복사해 만들고 리드 이름을 채운다 (루트 README 진행 현황 표 참조).
본인 섹션(`### <이름> — <개념>`)에 고른 개념 / 내용 리마인드 / 나에게 어떤 의미였나를 채운다. LinkedIn 칸은 `(발행 후 링크)`로 둔다.
**다른 참여자의 섹션은 절대 수정·삭제하지 않는다.**

### 7. LinkedIn 초안 (로컬 전용)
`chapters/chNN/<이름>/_drafts/linkedin-draft.md`에 작성. 톤 규칙 (루트 README 준수):
- 대괄호 제목 금지, 정제된 존댓말, 번호 리스트로 구조화, 해시태그 금지, em dash(—) 금지
- 개념 설명과 본인 경험·적용을 한 덩어리로
- 말미에 "발행 후 링크를 챕터 README에 기록" 체크리스트 한 줄

### 8. 배포 (정리본만)
1. `git status --short`로 변경 파일 확인. `_drafts/` 경로가 목록에 보이면 **멈추고** gitignore부터 고친다.
2. 스테이징은 명시 경로만: `chapters/chNN/` 하위 정리본·README (+ 필요 시 `.gitignore`). `git add -A` 금지.
3. `git diff --cached --stat`을 사용자에게 보여주고 커밋·푸시한다. 커밋 메시지 예: `docs(ch02): 임정 2장 정리 + 기본글`
4. **검증**: 푸시 후 `git ls-files chapters/chNN/<이름>/`에 `_drafts` 파일이 0건인지 확인하고 결과를 보고한다.

### 9. 요약 출력
- 푸시된 파일 / 로컬에만 남은 파일 목록
- 선택한 핵심 개념과 이유
- 남은 일: LinkedIn 발행 → 링크를 챕터 README 본인 섹션에 기록 (다음 커밋에서)
