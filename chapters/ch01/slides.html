§1 에이전틱 코딩이란

AI가 자율로 수행한다

기존 "답변 AI"와 무엇이 다른가

  • 클로드 코드 = 파일 시스템 탐색 · 코드 수정 · 테스트 실행까지 AI가 자율로 수행하는 접근 방식
  • 공식 워크플로: 탐색 → 계획 → 구현 → 커밋
  • 이전 AI: 사용자 질문에 답변 · 에이전틱 AI: 스스로 문제 분석 + 도구 자율 활용

§5 전략 1·2

컴팩션 & 클리어

  • 컴팩션(Compaction) — 대화를 요약. 단, 요약이므로 컨텍스트 유실 가능성 항상 존재
  • 클리어(Clear) — 대화를 버리고 새 컨텍스트에서 재시작. 방향이 잘못됐다고 판단되면 클리어가 더 낫다
  • 클리어와 함께 Memory.md도 정리 — 과거의 잘못된 지식이 남아 있으면 클로드가 계속 오답을 내놓는다
§5 전략 3 — ★ 가장 중요

점진적 공개(Progressive Disclosure)

  • 모든 파일을 다 읽으면 토큰 낭비
  • 클로드는 파일에 frontmatter(가벼운 요약)를 미리 작성해 둔다
  • 요약만 보고, 읽어야 하는 순간에만 본문을 읽는다 — 불필요하면 본문 접근 ✗
흐름
frontmatter 읽기 → 필요 여부 판단 → 필요 시 본문 읽기 → 불필요 시 스킵
§5 전략 4

서브에이전트 활용

  • 독립 컨텍스트를 가지며 병렬로 여러 작업 수행 가능
  • 서브에이전트 간 컨텍스트 직접 공유 불가 → 공통 메모 파일로 우회
  • 동시에 같은 파일에 쓰면 동시성(충돌) 이슈 발생
  • 누군가 파일을 점유 중이면 쓰기를 기다리도록 제어 필요
메모 쓰기 스킬에 lock 지원을 보완하는 것이 중요하다.
§5 전략 5

구조화된 메모 작성

  • 전략 3(점진적 공개) + 전략 4(서브에이전트 공유)와 연결
  • 메모를 구조화해 두면 재참조·공유 비용이 줄어든다
  • frontmatter가 잘 갖춰진 파일 = 점진적 공개의 기반
구조화 → frontmatter 요약 → 점진적 공개 → 토큰 절약
구조화 → 공통 메모 파일 → 서브에이전트 협업
§6 스킬 프론트매터 필드

프론트매터 — 핵심 필드

필드 필수 여부 설명
nameNo스킬 목록에 표시될 이름. 미지정 시 디렉터리명.
descriptionRecommendedClaude가 스킬 적용 시점 판단에 사용. 첫 문단이 기본값. 설명+when_to_use 합산 1,536자 제한.
when_to_useNo스킬 호출 트리거 문구 · 예시 요청. 1,536자 cap에 포함.
argument-hintNo자동완성 힌트. 예: [issue-number]
argumentsNo위치 인수 이름 → $name 치환. 공백 구분 문자열 또는 YAML 리스트.
disable-model-invocationNotrue 시 Claude 자동 로딩 차단. /name 수동 호출 전용.
allowed-toolsNo이 스킬 활성 중 권한 없이 사용 가능한 도구 목록.
disallowed-toolsNo이 스킬 활성 중 제거할 도구. 다음 메시지 시 복원.
modelNo스킬 활성 중 사용할 모델. 다음 프롬프트 시 세션 모델 복귀.
contextNofork 설정 시 포크된 서브에이전트에서 실행.
hooksNo이 스킬 생명주기 범위의 훅.
pathsNo특정 파일 패턴 매칭 시에만 자동 활성.
중요 — 프론트매터를 별도 파일로 정의하지 않으면 존재하지 않는 argument를 잘못 사용할 수 있다. 반드시 global 파일로 저장해 사용.
§7 클로드 코드 구성요소

6가지 구성과 Skills 통합

  • 구성: commands agents skills hooks output-styles settings.json
  • commands는 deprecated — 클로드는 skills로 통합. commands 더 이상 사용 안 함
  • skills 자체에서 agent · hook 별도 지정 가능
  • 패키지 구조를 스킬 내 서브스킬로 재활용 가능
commands ← deprecated → skills (통합)
§8 비대화형 세션

claude -p / SDK 자동화

  • claude -p, Claude SDK로 비대화형 세션 실행 가능
  • 권한에 부딪히면 응답 없이 무한 대기 → 권한 관리가 핵심
  • --dangerously-skip-permissions 플래그로 권한 스킵 가능
주의 — 플래그 이름 그대로 위험. 신뢰된 환경·격리 환경에서만 사용할 것.
§9 rewind (체크포인트)

체크포인트로 되돌아가기

  • rewind 옵션으로 체크포인트로 이동 — 잘못된 메시지 입력 후 컨텍스트 초기화에 유용
  • 체크포인트는 해당 대화가 마무리될 때 갱신
  • 클로드의 Edit, Write 작업만 기록
  • Bash 셸 · API 호출로 인한 변경은 되돌릴 수 없다
